# 喵 · 桌面伙伴 使用说明书

> 最近更新：2026-07-01

---

## 1. 这是个啥

一只浮在 macOS 桌面上的 Live2D 伙伴 —— 有脸、会陪你、记得住你，还能连上你的 App
和网页帮你干活。本质是一个**带身体的 LLM agent**：能调用工具、读屏、记得你说过的话、
主动搭话；自己干不了的重活会委派给后台同事（Codex / Claude / Hermes）。

定位一句话：和社交软件相反的「不社交」桌面伴侣 + 工作助手 —— 没有 feed / 关注 / 算法，
只属于你一个人，默认全在本地。

---

## 1.5 首启与语言

**第一次打开**会弹「认识喵」引导九宫格（环境/权限/大脑/嘴/耳朵/眼睛/搜索/形象），配好
「大脑」（聊天模型）就能用，其余可以以后在设置里慢慢解锁。走完/跳过后，**右键 →「重新
引导…」**随时能回看。

**界面语言**：首次安装时**跟随 macOS 系统语言** —— 中文系统显示简体中文，其它语言显示
英文。想手动切：**设置 → 通用 → 语言**，选「简体中文 / English」→「应用并重启」生效。
一旦手动选过就固定下来，不再跟系统变。喵的**聊天回复语言**则始终跟随 macOS 系统语言
（与界面开关解耦）。

## 2. 安装 / 启动 / 退出

**安装**：去 [Releases](https://github.com/solaswing/avatar-releases/releases/latest) 下最新的 `Mao-x.y.z.dmg` → 打开 DMG → 把里面的 App 拖进「应用程序」。已 Developer ID 签名 + Apple 公证，干净 Mac 双击即用，**不会有 Gatekeeper 警告**。

**启动**：从启动台 / 应用程序双击打开。第一次会弹「认识喵」引导（见 §1.5）；想开机自启，设置 → 通用 → 登录自启。

**退出**：菜单栏 → 退出喵（⌘Q）。

## 3. 交互入口（5 种）

| 操作 | 效果 |
|---|---|
| **左键单击** 数字人 | 弹出对话气泡 + 输入框 |
| **右键** 数字人 | 上下文菜单（设置/隐藏/切换角色/...） |
| **拖文件** 到数字人或灵动岛 | 询问你想对文件做什么（发邮件/微信/翻译/整理/**压缩**，拖压缩包则问要不要**解压**...） |
| **按住右 Option (⌥)** | Push-to-talk 语音输入，松开即转文字 |
| **⌃⌥S 截图快捷键**（可在设置改） | 框选一块（截图时按空格切**整窗口**）→ 弹一排动作：复制/保存/读文字OCR/识别二维码/标注/钉在桌面/截窗口（见 §8 截图）|

输入框里支持标准编辑快捷键：⌘C / ⌘V / ⌘X / ⌘A / ⌘Z / ⇧⌘Z

## 4. 两种显示模式

### 主体模式（默认）
- Live2D 角色浮在桌面上，可拖动
- 鼠标在角色 silhouette 内时窗口不穿透，外部空白处穿透
- 头顶/身侧出气泡显示对话

### 灵动岛模式
- 紧贴 MacBook notch 的迷你 pill
- 右键数字人 → **隐藏** 进入岛模式
- 鼠标 hover 自动展开成播放器面板（专辑封面/标题/艺人/歌词/进度条）
- 点击岛 → 还原为主体模式

## 5. 角色 + 人格

### 角色（决定脸 + 默认语音 + 性格）

**默认就是喵这张脸。** 角色系统是**数据驱动**的——在 **设置 → 角色设定** 里可以
**新增 / 删除 / 编辑**自定义角色：每个角色绑一张脸（形象）+ 默认音色 + 性格设定。
角色之间随时切：右键 → 选角色。

> 形象可在角色设定里导入你自己的 Live2D 模型；正式分发版只内置官方示例脸「喵」
> （Live2D 授权所限，其它脸不随分发包带）。

### 人格（决定主动行为 + 说话风格）
- **闲聊** (默认)：自然亲切、会主动搭话
- **工作**：克制、不闲聊、只回你直接问的
- **安静**：除非你主动叫她，否则一律不出声（只在气泡里回）

## 6. 大脑：对话发给哪个模型

每个角色在 **设置 → 角色设定** 里绑一个聊天模型，你这条对话就发给它。**她每轮都拿着全套
工具、自己决定要不要调** —— 你正常说话就行，不用点名工具。自己干不了的重活（写代码、跑你
配好的工作流、要很多步/很深的研究）会**委派给后台同事**（见 6.5），不硬撑。

主动场景（主动搭话、每日热点日报这类）用一个单独的**辅助/后台模型**（设置 → Agent 设置），
跟你聊天的主模型分开，省钱。

**能选的模型**（设置 → 角色设定 / API 管理）：
- **本地**：MLX（Apple Silicon 原生，可离线）/ Ollama 本地（自动列出你 `ollama pull` 过的）
- **云端**：ofox 通道、Ollama 云、Agnes、Gemini（Google 的 OpenAI 兼容端点）
- **自定义 API**（任意 OpenAI 兼容服务）：设置 → API 管理 → 顶部「＋ 添加 API」，填 名称 +
  Base URL（`https://host/v1` 即可）+ API Key，点「更新模型列表」拉取并探活，模型即进角色
  选择器。OpenRouter / DeepSeek 官方 / 硅基流动 / 本地 vLLM… 都能这么接，零改代码。

> 拉模型列表时会顺手探活：不可用 / 要会员（403/402）的模型自动隐藏，不会让你选到点不动的。

## 6.5 委派给同事（干不了的活交给谁）

数字人当"前台"，活儿超出她自带工具时，会交给一个更专业的后台 agent 去跑。能交给三家同事：

| 同事 | 适合干啥 | 怎么算"在线" |
|---|---|---|
| **Hermes** | 你提前配好的工作流（写专报、查资料库、做调研稿件这类） | 本机要有 Hermes 任务服务在跑（127.0.0.1:8642） |
| **Claude** | 通用强 agent：写代码、写长文、深度研究、复杂分析 | 配好委派 token（见下） |
| **Codex** | 同类通用强 agent，尤其擅长写代码、改仓库 | 终端登录过（`codex login`） |

**怎么挑谁**：你可以直接点名——"这个交给 Claude"、"让 Codex 跑"。不点名的话，她卡住/判断超纲时会**按任务类型自动挑最合适、且当前能用的那个**：代码类优先 Codex / Claude，工作流类优先 Hermes，其它（研究/分析/写作）优先 Claude；要是首选没在线，就顺位换下一个能用的。

**永远先问你一句**：委派前一定弹个"这个交给 X 跑行吗"让你点头，点了才真跑。要是目标没连上 / 没装 / 没登录，她会**老实说交不出去、也别假装能干**——不会瞎答应。

**两种力度**：
- **只读（默认）**：只查资料、动脑、写文字，绝不碰你的文件。
- **写入（write）**：能在你指定的某个目录里改文件、跑命令——更强，也更险，所以要显式开。

### 给三家同事配登录 / token

委派要真能跑，得先让对应同事"上线"。一次配好，之后就一直能用：

- **Hermes**：保证本机的 Hermes 任务服务在跑就行（健康检查走 `:8642/health`），app 这边不用填东西。
- **Claude**（要在 app 里填一段 token，因为它钥匙串里的登录传不进子进程）：
  1. 终端跑一次 `claude setup-token`（绑你的 Claude 订阅，**免费、不按量计费**）。
  2. 浏览器里授权 → 它给你一串"授权码"，**粘回终端**、回车。⚠️ 这串授权码是给终端的，**不是**填进 app 的。
  3. 终端这才吐出 `sk-ant-oat01-…` 开头的**长串**（上百个字符）→ 复制整条，填进 **设置 → 外部智能体 → 「Claude 委派」卡** 的输入框，保存。（落盘在 `~/.config/desktop-avatar/claude.json`。框里会校验：贴成短的授权码、或只贴了半截，都会当场提示你。）
- **Codex**：终端跑一次 `codex login` 就行，登录信息存在 `~/.codex/auth.json`，**app 这边不用再填**。

小结：三家里只有 **Claude 需要在设置里填 token**；Codex 终端登录过即可，Hermes 只要服务在跑。

## 7. 语音

### 输入：PTT（Push-to-Talk）
- 按住右 ⌥ → 录音中（岛/气泡显示波形）
- 松开 → SFSpeechRecognizer 转文字 → 自动发送
- 首次使用会要授权：麦克风 + 语音识别

### 输出：TTS 引擎切换
设置 → 角色 → 语音引擎：
- **Edge TTS**（默认，免费，WebSocket 直连 Microsoft）
- **火山 Doubao**（需配 OpenAPI key，音质显著更好）
- **ElevenLabs**（需 key，高质量情感音色）
- **MeloTTS / Kokoro**（本地 server，备选）

> 完整引擎清单与配置见 §15 语音引擎。

每个角色可单独绑音色 + 调速度/音调/音量。

### 字幕同步
文字按音频时长**逐字 drip**出现，跟语音同步（默认约 0.07 秒 / 字）。

## 9. 文件拖拽流（新）

1. 从 Finder 拖一个/多个文件 → 数字人主体或灵动岛
2. 数字人接到 → 高亮橙色边框 → 弹气泡："想让我对它做啥呀？"
3. 你说/打 → "发我邮箱"、"翻译"、"整理"、"发到企业微信"、"**压缩/打包**"...
4. 自动调对应工具完成

**压缩 / 解压都会先问一句再动手**（喵功能多，拖进来 ≠ 一定要解/压）：

- 拖 **压缩包**（zip/tar.gz 等）→ 问"要我解压吗？"→ 你回"解压/好"才解到同级新文件夹并打开（7z/rar 会提示装 Keka）。
- 拖 **文件 / 文件夹** → 开放问询里"压缩打包"是其中一个选项 → 你回"压缩/打包"才压成 zip（单个保留原名 `foo.zip`，多个叫 `打包.zip`）放同目录、访达里选中。
- 拖 **图片** → 直接当作这一轮的看图输入（接着问它"这是什么/翻译一下"）。
- 拖 **表格**（csv/xls/xlsx）→ 用 pandas 精确分析（见 `query_table`）。

90 秒不回应自动清空，不会"占用"后续对话。

## 10. 主动行为（Presence Engine）

不打扰人但留意你。各触发器都有独立冷却防止打扰：

| 触发条件 | 反应 |
|---|---|
| 你复制了网址（2 秒级感知） | 问"要我帮你打开吗" |
| 30+ 分钟离开后回来 | 打招呼 |
| 一个文件夹/IDE 卡住很久 + 闲置 | 问要不要喘口气 |
| 听了一首很伤的歌 | 关心一下 |
| 邮件来了未读 | 提醒一下 |
| 日历事件快到 | 提前提醒 |
| 早 7 点首次活跃 | 道早安 + 天气 |
| 长时间窗口不动 | 提示 |

**安静模式下全部关闭**。聊天/说话中不打断。

## 10.5 每日热点日报

喵能抓全网热点（微博/抖音/小红书/知乎/头条/百度…），**跨平台并簇、按印证度排序**（多个
平台都在说的排前面），再用喵的口吻润色成一份干净日报念给你。

- **随口问**："今天有啥热点 / 来份日报" → 当场生成。
- **定时自动播**：**设置 → 订阅管理 → 每日日报** 打开开关、设个时间（如每天 9:00），到点
  自动播报。源没取到会走上次缓存并**显式告诉你是旧的**，不硬编。
- 念的时候不读网址（URL 只在气泡里留可点）。

## 11. 学习能力（越用越懂你）

喵是**边用边学**的，学到的东西可在设置里逐条看 / 逐条删（可控）：

### 操作小窍门（per-App）
她操作某个 App **撞了坑又自己克服**之后，会自动记一条该 App 的小窍门（比如"搜歌先点全网
搜"、"播放键 AX 读不到要用视觉点"），下次操作同一个 App 自动带上。你也能直接说"记一下这个
App 的窍门"。在 **设置 → 自主学习** 里能看到 / 删除每条，也能关掉「自动反思」。

### 长期事实（关于你）
你说的"记住我喜欢喝美式"这类，她记进长期事实，下次相关时才会用上。在 **设置 → 记忆** 里
可查 / 删。

### Shortcuts.app 桥接
能通过 LLM 直接调用你 Shortcuts.app 里的快捷指令（`list_shortcuts` / `run_shortcut`）。

> 注：早期那套"录制流程→回放"的教学模式已下线（AX-blind 下不可靠）；现在走的是上面这种
> "撞坑笔记 + 事实"的渐进学习，不是录屏复刻。

## 12. 记忆系统（两层）

| 层 | 存哪 | 行为 |
|---|---|---|
| **短期** | 内存里的会话缓冲（最近几轮，含工具结果） | **退出 app 就丢**，从不落盘 |
| **长期** | `facts-store.json`（+ `facts.txt` 镜像，向量在 `fact-embeddings.jsonl`） | 只记**你自己说的**事实，自动入库；可手动删 / 纠正替换 |

- 长期事实只从**你的话**里抽（不会把喵自己的人设台词当成你的偏好），且要过门槛才记。
- 召回以**关键词**为主、向量兜底（避免"某个词对每个问题都排第一"那种糊）。
- 在 **设置 → 记忆** 里逐条查 / 删；右键 → **清空记忆** 一键归零。

> 早期那套 `memory.jsonl` / `summary.txt` / `memory-tree.json`（原文+摘要+话题树）已在
> 2026-06 的记忆重构里删掉，换成上面这个更干净的两层模型。

## 16. 配置文件路径

凭据/数据都在用户目录（目录 0700、敏感文件 0600），**不在仓库里**：

```
~/.config/desktop-avatar/
├── ofox.json                ofox 凭证
├── custom-providers.json    自定义 API（名称/Base URL/key）+ custom-models.json（拉到的模型列表）
├── volcengine-tts.json      火山 TTS 凭证
├── volcengine-asr.json      火山流式 ASR 凭证
├── volcengine-openapi.json  火山 OpenAPI（用量查询）
├── ollama-cloud.json        Ollama 云
├── ollama.txt               Ollama 本地兜底模型名
├── qqmusic.json             QQ 音乐 API key
├── claude.json              Claude 委派 token（claude setup-token 拿到的长串）
├── facts-store.json         长期事实主存（带 salience/软删/纠正替换）
├── facts.txt                facts-store 的派生镜像（+ fact-embeddings.jsonl 向量）
├── digest-cache.json        每日热点日报缓存（源没取到时兜底）
├── app-knowledge.json       各 App 的用途 + 操作小窍门
├── operation-memory/        每个 App 怎么点的"配方"
├── mail-history.jsonl       检测到的验证码（邮箱 + 短信）
├── usage.jsonl              用量记录
└── tool-history.jsonl       工具调用历史

~/.hermes/.env             Hermes API key（Hermes 委派用）
~/.codex/auth.json         Codex 登录信息（codex login 写入，Codex 委派用）
```

> 界面语言不存在这儿、而在 UserDefaults（`DesktopAvatar.appLanguage.v1`）；没设过 = 跟系统。

## 19. 已知边界

- 截图 / 读屏走 ScreenCaptureKit（`SCScreenshotManager`），需「屏幕录制」权限
- **操作能连多少步，主要看你给它配多强的"大脑"**：配个强模型，常见操作（开 App、搜歌放歌、搜番点开、填表单）能稳稳连着跑、还**越用越熟**——每次成功都攒一条该 App 的操作配方，下次更快更稳；本地小模型则短一些。真正复杂、通用操作够不着的，有两条路：**① 委派给专业 agent**（Codex / Claude / Hermes，见 §6.5）；**② 给它定制一个专门工具 / MCP**（把那套复杂流程固化成一个工具，以后一句话调用，见 §15「MCP 接入」/ `install_tool`）
- QQ Music 不响应 HID 媒体键，所以远程控制走 AppleScript / CGEvent
- Edge TTS 是 Microsoft 公开端点，Microsoft 偶尔改 auth token；挂了会自动 fallback Volcengine 或 MeloTTS
- 一句话回复可能要 ~600-900ms 才出首音（LLM SSE + TTS 启动）
